# Scorecard de Evaluación de Agentes

## Escala

- 2: cumple claramente
- 1: cumple parcial
- 0: no cumple

## Planner

- MVP acotado.
- No inventa backend.
- Hitos verificables.
- Pruebas realistas para Phaser.
- Riesgos y mitigaciones útiles.

## Test Writer

- Prioriza lógica testeable.
- Casos de victoria/derrota incluidos.
- Propone estructura de tests concreta.
- Evita tests frágiles del motor.
- Verificación clara.

## Implementer

- Cambios mínimos.
- Respeta alcance.
- El juego queda ejecutable.
- La lógica clave queda separada.
- No introduce deuda obvia.

## Diagnoser

- Resume fallo con precisión.
- Identifica causa raíz.
- Propone fix mínimo.
- Añade prueba de regresión.
- Da comando de verificación.

## Reviewer

- Detecta riesgos reales.
- Prioriza bien.
- Usa evidencia.
- Señala gaps de testabilidad.
- Propone remediación accionable.

## Debater

- Presenta al menos dos opciones.
- Expone trade-offs reales.
- Recomienda una.
- Justifica impacto técnico.
- Mantiene el MVP acotado.
