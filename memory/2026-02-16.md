## Dev UX fixes (CORS + usuarios CMS)
- CORS: el backend solo habilita CORS si existe `CORS_ALLOW_ORIGINS` (ver `src/app/cors.py`). Para dev se usó:
  - `CORS_ALLOW_ORIGINS='http://localhost:5173,http://127.0.0.1:5173,http://192.168.0.158:5173'`
- Seed dev: se actualizó `src/app/seed.py` para crear usuarios CMS si no existen:
  - `agent@a.com / secret123` (role `agent`)
  - `admin@a.com / secret123` (role `admin`)
  - (importó `select` de SQLModel)
  - `pytest -q` pasó tras el cambio.
- Importante: se detectó confusión porque el backend a veces se levantó **sin** `SEED_DEV=1`, causando 401 `INVALID_CREDENTIALS` aunque el usuario "debería" existir. Al levantar con `APP_ENV=dev SEED_DEV=1` se confirmó que en la DB existen: `a@a.com` (visitor), `agent@a.com` (agent), `admin@a.com` (admin) y el login funciona.

- Problema: URLs seed apuntaban a `https://img.local/...` → `ERR_NAME_NOT_RESOLVED`.
- Fix backend:
  - Se montó estático `StaticFiles` en `main.py` para servir `UPLOAD_DIR` en `/uploads`.
  - Se ajustó default `PUBLIC_UPLOAD_BASE_URL` en upload CMS a `http://localhost:8000/uploads`.
  - En `init_db(seed=True)` se escriben PNGs placeholder 1x1 en `./uploads/prop1_*.png` y `./uploads/prop3_*.png` para que el frontend renderice fotos seed.
- Nota: hubo un crash inicial por `./uploads` inexistente; se ajustó a crear el dir antes y usar `check_dir=False`.
