### Proyecto: Sprite Atlas Builder
Herramienta web para construir/cortar sprite sheets para Tiled.

**Ubicación:** `/home/administrador/.openclaw/workspace/projects/sprite-atlas-builder/`

**Stack:** React + Vite + TypeScript (frontend), FastAPI (backend skeleton)

**Modes:**
- **Builder Mode:** Carga imágenes individuales, acomoda en grid, exporta PNG + JSON
- **Tiled Cutter:** Corta tiles de spritesheets existentes, exporta PNG + .tsx para Tiled

**Estado:**
- Build exitoso
- Dev server: `npm run dev` (puerto 3000)
- 31 tests unitarios passing
- Hito: MVP funcional con Grid Mode

**Usage:**
```bash
cd /home/administrador/.openclaw/workspace/projects/sprite-atlas-builder
npm run dev
# Abrir http://localhost:3000
```

## Agentes Red Team / Blue Team

Basado en BlueCodeAgent de Microsoft Research, se crearon dos nuevos agentes:

### Nuevos agentes
| Agente | Rol | Propósito |
|--------|-----|-----------|
| `code_critic` | 🔴 Red Team | Criticar código, buscar fallos, edge cases, vulnerabilidades |
| `defender` | 🔵 Blue Team | Responder a críticas con soluciones concretas |

### Ubicación
- `~/.openclaw/workspace/ollama-agents/agents/code_critic.yaml`
- `~/.openclaw/workspace/ollama-agents/agents/defender.yaml`
- Pipeline: `~/.openclaw/workspace/ollama-agents/pipeline/run_redblue.py`

### Flujo propuesto
```
IMPLEMENTER → CODE_CRITIC (Red) → DEFENDER (Blue) → IMPLEMENTER (fix)
```
